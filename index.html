cvsCamCtrl Library

CREVIS cvsCamCtrl 라이브러리의 API, 자료형, Feature 조회, 확장 레지스터 기능 안내

Version: 1.2.0Date: 2026. 3. 12

1. 개요#

cvsCamCtrl는 CREVIS에서 제공하는 가상 프레임그래버 라이브러리입니다. 본 문서는 첨부된 cvsCamCtrl.h 헤더를 기준으로 API, 상수, 구조체, 열거형, Feature 메타데이터 조회 기능, 모듈 기반 확장 레지스터 접근 기능까지 포함하여 정리한 프로그래머 가이드입니다.

1.1 문서 기준#

  • 헤더 우선 원칙: 함수 시그니처, 타입, 기본값, enum/struct 정의는 첨부된 cvsCamCtrl.h 선언을 우선 기준으로 삼았습니다.

대표적인 기본 사용 순서는 다음과 같습니다.

ST_InitSystem();
ST_UpdateDevice();            // 또는 ST_UpdateDeviceWithTimeout()
ST_OpenDevice(...);
ST_InitBuffer(...);

/* 연속 취득 */
ST_AcqStart(hDevice);
ST_GrabImage(...);           // 또는 ST_GrabImageAsync(...)
ST_AcqStop(hDevice);

/* 단발 취득 */
ST_SingleGrabImage(...);

ST_FreeBuffer(...);
ST_CloseDevice(hDevice);
ST_FreeSystem();

2. 정의 및 상수#

이 섹션에서는 이벤트 ID, 오류 코드, 장치 정보 상수, XML 버전, 가시성 수준, 모듈 식별 매크로, 색상 변환 코드 조합용 비트 필드를 설명합니다.

2.1 이벤트 정의#

이벤트는 Grab 이벤트와 장치 이벤트로 구분됩니다. Grab 이벤트는 이미지 획득 흐름에서 발생하고, 장치 이벤트는 장치 상태나 인터페이스 상태 변화에서 발생합니다.

2.1.1 Grab 이벤트#

이름 값 설명
EVENT_NEW_IMAGE 0x0 새 이미지가 정상적으로 수신됨
EVENT_GRAB_ERROR 0x1 Grab 과정에서 오류 발생
EVENT_GRAB_TIMEOUT 0x2 지정 시간 내 Grab이 완료되지 않음

2.1.2 장치 이벤트#

일부 이벤트는 특정 인터페이스에서만 지원됩니다. 헤더 주석 기준 인터페이스 범위는 GEV, U3V 또는 CXP 관련 속성으로 구분됩니다.

이름 값 설명 비고
EVENT_DEVICE_DISCONNECT 0x0010 장치 연결 해제 감지 GEV, U3V
EVENT_EXPOSURE_END 0x0014 노출(Exposure) 종료 GEV, U3V
EVENT_ACQUISITION_TRANSFER_START 0x0018 이미지 데이터 전송 시작 GEV
EVENT_ACQUISITION_TRANSFER_END 0x001C 이미지 데이터 전송 종료 GEV
EVENT_OVER_TRIGGER 0x0020 과도한 트리거 감지 GEV
EVENT_OVER_TRIGGER_START_MISSED 0x0024 트리거 시작 이벤트 누락 감지 U3V
EVENT_OVER_TRIGGER_END_MISSED 0x0028 트리거 종료 이벤트 누락 감지 U3V
EVENT_TRIGGER_START 0x002C 트리거 시작 감지 U3V
EVENT_WARNING_TEMPERATURE_STATE 0x0030 온도 경고 상태 U3V
EVENT_CRITICAL_TEMPERATURE_STATE 0x0034 온도 위험 상태 U3V

5.1 콜백 함수 정의#

라이브러리에서 제공하는 콜백 함수 형식입니다.

5.1.1 GrabCallback#

Grab 이벤트가 발생할 때 호출되는 콜백 함수 형식입니다.

형식

typedef void (WINAPI* GrabCallback)(int32_t eventID, const CVS_BUFFER* pBuffer, void* pUserDefine);

매개변수

이름 입출력 타입 설명
eventID IN int32_t 발생한 Grab 이벤트 ID입니다.
pBuffer IN const CVS_BUFFER* Grab된 이미지 버퍼 포인터입니다.
pUserDefine IN void* 사용자 정의 데이터 포인터입니다.

반환값

  • 없음 (void)

예제

void WINAPI OnGrab(int32_t eventID, const CVS_BUFFER* pBuffer, void* pUserDefine)
{
    if (eventID == EVENT_NEW_IMAGE && pBuffer != nullptr)
    {
        printf("frame: %d x %d, blockID=%llu\n",
               pBuffer->image.width,
               pBuffer->image.height,
               (unsigned long long)pBuffer->blockID);
    }
}

5.1.2 EventCallback#

장치 이벤트가 발생할 때 호출되는 콜백 함수 형식입니다.

형식

typedef void (WINAPI* EventCallback)(const CVS_EVENT* pEvent, void* pUserDefine);

매개변수

이름 입출력 타입 설명
pEvent IN const CVS_EVENT* 장치 이벤트 정보 구조체 포인터입니다.
pUserDefine IN void* 사용자 정의 데이터 포인터입니다.

반환값

  • 없음 (void)

예제

void WINAPI OnDeviceEvent(const CVS_EVENT* pEvent, void* pUserDefine)
{
    if (pEvent == nullptr)
        return;

    printf("eventID=%d, timestamp=%llu, payload=%u bytes\n",
           pEvent->id,
           (unsigned long long)pEvent->timestamp,
           pEvent->dataLength);
}

);

{

pBuffer IN CVS_BUFFER* 해제할 버퍼 구조체 포인터입니다.

반환값

  • MCAM_ERR_OK: 성공
  • 그 외: 오류 코드

비고

  • 해제한 버퍼를 다시 사용하려면 ST_InitBuffer()를 다시 호출하십시오.

예제

int32_t hDevice = /* ST_OpenDevice() 성공 후 얻은 핸들 */ 0;
CVS_BUFFER buffer = {};

if (ST_InitBuffer(hDevice, &buffer, 0) == MCAM_ERR_OK)
{
    CVS_ERROR err = ST_FreeBuffer(&buffer);
    if (err != MCAM_ERR_OK)
    {
        printf("ST_FreeBuffer failed: %d\n", err);
    }
}

6. 주의사항#

  • 본 문서는 첨부된 헤더 선언을 기준으로 정리되었습니다. 실제 동작 세부사항은 배포된 DLL/SDK 버전에 따라 달라질 수 있습니다.
  • 일부 이벤트와 장치 정보는 인터페이스(GEV, U3V, CXP)에 따라 제공 여부가 달라질 수 있습니다.
  • 문자열/버퍼 인수는 호출 전에 충분한 크기의 메모리를 확보하고, pSize 계열 인수는 버퍼 크기와 실제 반환 크기를 함께 처리하도록 작성하는 것이 좋습니다.
  • ST_GrabImage(), ST_GrabImageAsync(), ST_SingleGrabImage(), ST_CvtColor() 사용 시 버퍼의 초기화와 메모리 해제 순서를 명확히 관리하십시오.
  • Feature 및 모듈 기반 API는 CvsModuleType 값과 대상 노드 이름이 정확해야 정상 동작합니다.
